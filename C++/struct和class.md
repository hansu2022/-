相同点：

- 如果结构体没有定义任何构造函数，编译器会生成默认的无参数构造函数。如果类没有定义任何构造函数，编译器也会生成默认的无参数构造函数。

从本质上讲，`class` 和 `struct` 在 C++ 中是几乎等同的，它们都可以定义成员变量、成员函数、构造函数、析构函数，也可以进行继承和多态。它们最核心的区别在于**默认的访问权限**。
- **默认成员访问权限：**
    - `class`：成员的默认访问权限是 **`private`**。
    - `struct`：成员的默认访问权限是 **`public`**。
        
- **默认继承方式：**
    - `class`：默认的继承方式是 **`private`**。
    - `struct`：默认的继承方式是 **`public`**。
这两个默认行为是 `class` 和 `struct` 在语义上唯一的不同。除此之外，它们是完全通用的。

POD 类型指的是与 C 语言兼容、没有复杂特性的数据类型。这使得它们可以安全地使用 C 语言的函数（如 `memcpy`）进行操作，或者在不同语言之间进行数据传递。
**​​**C++11/14/17 标准中的定义：** 现代 C++ 标准将 POD 拆分成了两个独立的特性，一个类型必须同时满足这两个条件才能被称为 POD：

1. **平凡性（Trivial）：**
    
    - 有平凡的默认构造函数，即不执行任何操作。
    - 有平凡的拷贝/移动构造函数和赋值运算符。
    - 有平凡的析构函数，即不执行任何操作。
    - **意义：** 平凡性保证了该类型的对象可以安全地通过 `memcpy` 进行复制，而不会引起任何副作用。
        
2. **标准布局（Standard Layout）：**

    - 所有非静态成员都具有相同的访问权限（`public`、`private` 或 `protected`）。
    - 没有虚函数或虚基类。
    - 所有基类也都是标准布局类型。
    - **意义：** 标准布局保证了该类型的内存布局是连续的，并且成员的顺序是确定的，这使其与 C 语言的数据结构兼容。

- **`struct`：** 通常用于定义**数据集合**。当你的类型主要是用来存储一组相关数据，并且这些数据没有复杂的行为或需要严格的封装时，使用 `struct` 是一个很好的选择。因为其成员默认为 `public`，代码会更简洁。
- **`class`：** 通常用于定义具有**行为和状态**的抽象概念。当你的类型需要封装数据、提供成员函数来操作数据，并且可能涉及复杂的逻辑、继承或多态时，使用 `class` 更符合面向对象的编程思想。它的成员默认为 `private`，这正好体现了**信息隐藏（Information Hiding）**的原则。
